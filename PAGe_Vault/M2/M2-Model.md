# M2 Model

## Purpose

M2 forecasts one- and two-week-ahead influenza positivity after ignition and alignment. It uses both live surveillance features and M1-derived alignment features.

## Inputs

- M1 alignment outputs: `newWeek`, `tau`, `delta`, `logit_f_eff`, peak timing estimates
- Raw current-season surveillance features derived prospectively: `z_ema` (Holt-smoothed logit positivity), growth rate
- Historical aligned training data for frozen or weekly-refit prediction

## Target

For horizon `h in {1, 2}`, M2 estimates future positivity:

- `Pr(positive at weekF + h | data observed through weekF, post-ignition, M1 state)`

## Statistical form

M2 is a joint binomial GAM fit over both horizons. Rows are stacked by lead so the model shares structure across `h = 1` and `h = 2` while allowing lead-specific smooths.

Key feature groups:

- **M1 template signal**: `logit_f_eff` — effective template positivity on the logit scale (`k_f` knots)
- **State signal**: `z_ema` — Holt-smoothed logit positivity (`k_e` knots)
- **Amplitude residual**: `z_resid = z_ema - logit_f_eff` — systematic deviation from the template (`k_r` knots)
- No rate-of-change covariate (`k_de = 0`): level-only Holt bias correction tracks rapid changes adequately

## Production spec (v12)

Best spec from v12 nested LOSO grid search:

| Parameter     | Value | Role                           |
|---------------|-------|--------------------------------|
| `k_f`         | 3     | template signal knots          |
| `k_e`         | 2     | state signal knots             |
| `alpha_state` | 0.20  | Holt EMA decay for `z_ema`     |
| `k_r`         | 2     | amplitude residual knots       |
| `k_de`        | 0     | rate-of-change term disabled   |
| `delta`       | 0     | no dilation term               |
| `Kr`          | 2     | amplitude residual flexibility |

- **LOSO NLL**: 33.49
- **GAM coefficients**: 25

## Bias correction

After prediction, a **level-only Holt EMA** corrects systematic residuals on the logit scale:

```
level <- alpha * (logit_obs - logit_pred) + (1 - alpha) * level
correction <- level    # applied by adding to raw logit prediction
```

- `alpha = 0.4` for both horizons
- Per-horizon (`h1`, `h2`) states maintained separately
- Bias resets on first post-peak transition to prevent rising-phase drift inflating post-peak forecasts

## Leakage rule

M2 is only valid if M1-derived features are built prospectively. Tuning uses nested LOSO: the held-out season's M1 covariates must be generated by an M1 model that never saw that season's templates.

## Deployment mode

Production uses **frozen GAM + sequential Holt bias correction**:

1. Load pre-trained GAM from `m2_production.rds`
2. Each week, update the bias level from newly observed positivity
3. Apply `level` correction to raw GAM logit predictions before output

Weekly refit mode is available (`refit_stage2_weekly`) but not the primary path.

## Output

- One- and two-week-ahead positivity predictions
- Optional prediction intervals
- Forecast conditioned on the current M1 alignment state
